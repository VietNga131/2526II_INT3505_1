# Instructions

## 1. Mục đích
Nghiên cứu, tìm hiểu và so sánh chuẩn OpenAPI (Swagger) với các ngôn ngữ/định dạng mô tả API phổ biến khác trên thị trường (API Blueprint, RAML, TypeSpec). Qua đó, hiểu rõ ưu/nhược điểm và triết lý thiết kế của từng chuẩn.

## 2. Yêu cầu
- Đưa ra bảng so sánh chi tiết các tiêu chí giữa OpenAPI và các công cụ tương tự.
- Xây dựng thư mục demo chứa ví dụ thực tế cho ứng dụng "Quản lý thư viện" bằng 4 chuẩn khác nhau.
- Demo việc tự động hóa: Sinh mã nguồn (Code Generation), sinh tài liệu (Doc Generation) hoặc sinh test tự động (Automated Testing) từ các file tài liệu.

## 3. Phương pháp so sánh
- So sánh dựa trên cú pháp (Ngôn ngữ thiết kế).
- So sánh dựa trên triết lý cốt lõi.
- So sánh dựa trên hệ sinh thái công cụ hỗ trợ (Tooling ecosystem).

## 4. Dàn ý chung
1. Tổng quan về các chuẩn mô tả API.
2. Bảng so sánh chi tiết (OpenAPI vs API Blueprint vs RAML vs TypeSpec).
3. Kết luận và Use-case (Khi nào nên dùng chuẩn nào).
